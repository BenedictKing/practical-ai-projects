## Layout Contract

- 页面采用单页、单栏、居中阅读布局；固定阅读顺序为：首屏定位 → 自我介绍 → AI 使用痛点 → 方法论/设计原则。
- 正文内容最大宽度为 `720px`；页面左右内边距为移动端 `24px`、桌面端 `32px`。
- 桌面端仍以单栏为主，不为“看起来更丰富”而强行改成双栏。
- 每个区块遵循同一结构：区块标签（可选） → 标题 → 说明文字 → 列表 / 原则项。
- 全页最多 1 个实心主 CTA；其余交互若存在，只能是文本链接。
- 区块之间优先使用留白而不是大面积背景色做分隔。
- Phase 1 不使用 sticky 导航、返回顶部悬浮按钮、侧边目录。
- Phase 1 不使用人物头像、项目截图、品牌墙、装饰性插图；表达重点是定位与结构清晰。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 细小图文间距、徽标与文字的最小贴合间距 |
| sm | 8px | 紧凑说明、标签与文本之间的间距 |
| md | 16px | 段落间距、默认组件间距 |
| lg | 24px | 区块标题与正文之间的间距 |
| xl | 32px | 同一区块内不同内容组之间的间距 |
| 2xl | 48px | 首屏文案组与 CTA 的间距、较大的内容断点 |
| 3xl | 64px | 页面主区块之间的间距 |

Exceptions: `44px` minimum height for CTA and any button-like link; `1px` borders/dividers are allowed and do not count as spacing tokens. Defaults chosen because upstream artifacts lock simplicity and readability but do not provide numeric spacing values.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 28px | 600 | 1.2 |
| Display | 40px | 600 | 1.2 |

Notes:
- 仅允许这 4 个字号与 2 个字重，避免 Phase 1 因排版层级过多而变乱。
- `Display` 只用于首屏核心定位标题；每页只出现 1 次。
- `Heading` 用于四个主区块标题；不要再引入额外的二级视觉标题体系。
- 正文每段控制在 2–4 行以内，优先短句表达，避免大段说明堆叠。
- Values are defaults inferred from `HOME-07` and项目“清晰、可信、可快速浏览”的原则；上游文档未提供更细的字号约束。

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | 页面背景、主要阅读表面 |
| Secondary (30%) | `#F8FAFC` | 轻分区背景、浅色卡片、区块衬底 |
| Accent (10%) | `#1D4ED8` | 主 CTA、文本链接、焦点环、当前锚点强调 |
| Destructive | `#DC2626` | 仅用于未来破坏性操作；Phase 1 不渲染此类操作 |

Accent reserved for: 首屏唯一主 CTA、正文中的真实链接文本、键盘 focus ring、当前锚点或小型区块标识。禁止把 accent 用作大面积背景、整段正文颜色或每个区块标题的主色。

Supporting tokens:
- Primary text: `#0F172A`
- Secondary text: `#475569`
- Border / divider: `#CBD5E1`

Notes:
- This palette is a default decision because upstream artifacts lock “可信、可快速浏览、不过度炫技” but do not specify brand colors.
- Light theme is the contract for Phase 1. Do not add dark mode in this phase.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 了解我的方法 |
| Empty state heading | 项目展示将在下一阶段加入 |
| Empty state body | 当前版本先聚焦我的定位、我关注的 AI 使用痛点，以及我如何拆解这些问题；具体项目入口会在下一阶段补充。 |
| Error state | 这部分内容暂时不可用，请刷新页面后重试；如果仍无法查看，请直接访问仓库 README 获取当前说明。 |
| Destructive confirmation | none — Phase 1 禁止出现删除、清空、覆盖、提交确认等破坏性操作，因此不设计确认弹窗 |

Tone rules:
- 用第一人称表达，但避免自夸式营销语气。
- 句子要具体，优先说明“我在解决什么问题”而不是“我有多热爱 AI”。
- 避免空泛词，如“赋能”“颠覆”“全栈 AI 革命”。
- 每个区块标题都应回答一个明确问题：我是谁、我在解决什么问题、这些问题为什么值得解决、我用什么原则判断方案是否有效。

Notes:
- `Primary CTA` is a default because Phase 1 需要唯一主行动，但上游文档未锁定具体按钮文案。
- `Empty state` 仅在执行者执意保留“后续模块占位”时使用；默认情况下，Phase 1 不应渲染空状态组件。
- `Error state` is a defensive default for static content fallback; this phase should avoid runtime data dependencies entirely.

## Interaction Contract

- 所有页内跳转都使用原生锚点行为；不要使用 scroll hijack、全屏分页滚动或自定义滚动容器。
- 主 CTA 使用实心 accent 样式；普通文本链接保持文本形态，并通过颜色与下划线表达可点击性。
- hover / focus 反馈只允许使用颜色、下划线、边框或阴影微调，过渡时长不超过 `150ms`。
- 键盘焦点必须始终可见，使用 `2px` accent focus ring 与 `2px` offset。
- 不使用 reveal-on-scroll、视差、数字滚动、自动切换内容、背景粒子、视频背景。
- 如果需要区块锚点，锚点目标必须是语义标题本身，而不是空白占位元素。
- 全页语义结构必须保持 `h1` → `h2` 的正常顺序；列表必须真的是 `ul/ol`，不能用纯 `div` 模拟。

Source: `REQUIREMENTS.md` HOME-07, `ROADMAP.md` Phase 1 success criteria, `01-CONTEXT.md` D-02 / D-03 / D-04.
