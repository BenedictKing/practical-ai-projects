# Roadmap: My Best Practice

## Overview

本路线图围绕 v1 的真实目标展开：先用单页主页建立清晰、可信、可快速浏览的统一入口，再把“我做过什么、解决了什么问题、该去哪里继续看”组织成可理解的项目展示，最后补上与主页内容同源的 README 精简版，让站内访客与 GitHub 访客都能在最短路径内理解定位与主要项目。

## Phases

**Phase Numbering:**
- Integer phases (1, 2, 3): Planned milestone work
- Decimal phases (2.1, 2.2): Urgent insertions (marked with INSERTED)

Decimal phases appear between their surrounding integers in numeric order.

- [ ] **Phase 1: 定位与主页骨架** - 建立清晰可信的单页入口，先讲清楚我在解决什么问题
- [ ] **Phase 2: 项目展示与继续浏览** - 让访客快速理解我做过什么，并进入具体项目或资料
- [ ] **Phase 3: README 同源精简版** - 让 GitHub 访客在仓库内也能快速理解定位与主要项目

## Phase Details

### Phase 1: 定位与主页骨架
**Goal**: 访客打开主页后，能快速理解作者定位、关注的 AI 使用痛点，以及整体信息结构
**Depends on**: Nothing (first phase)
**Requirements**: POSI-01, HOME-01, HOME-02, HOME-03, HOME-05, HOME-07
**Success Criteria** (what must be TRUE):
  1. 访客打开站点后，可以访问一个可正常浏览的单页主页，并在首屏快速理解定位为“专注解决 AI 使用中的实际阻碍”。
  2. 首页包含自我介绍或个人定位模块，访客可以快速看懂作者是谁、主要关注哪类 AI 使用问题。
  3. 首页包含 AI 使用痛点总结模块，能明确说明作者持续解决的是哪些真实使用障碍。
  4. 首页包含方法论或设计原则模块，且整页以清晰表达和可读性为优先，不依赖复杂视觉动效来传达信息。

**UI hint**: yes

### Phase 2: 项目展示与继续浏览
**Goal**: 访客可以浏览多个项目，并顺着清晰入口继续查看具体项目或补充资料
**Depends on**: Phase 1
**Requirements**: POSI-02, POSI-03, HOME-04, HOME-06, PROJ-01, PROJ-02, PROJ-03, PROJ-04
**Success Criteria** (what must be TRUE):
  1. 首页提供多个项目卡片或区块，作为继续浏览的主要入口。
  2. 开源访客、潜在客户和招聘方都可以通过项目展示快速理解“我做过什么”，并判断哪些项目与自己相关。
  3. 每个项目展示项都至少说明面向谁、解决了什么痛点，访客无需额外跳转即可做初步判断。
  4. 每个项目展示项都提供 GitHub 地址或在线访问地址，同时首页提供 GitHub / 文档入口，访客可以继续进入具体项目或补充资料。
**Plans**: TBD
**UI hint**: yes

### Phase 3: README 同源精简版
**Goal**: GitHub 访客无需进入主页，也能快速理解定位与主要项目，并看到与主页同源的精简内容
**Depends on**: Phase 2
**Requirements**: RDME-01, RDME-02, RDME-03
**Success Criteria** (what must be TRUE):
  1. 仓库中提供一个可独立阅读的 README 框架，作为主页内容的精简版。
  2. GitHub 访客仅通过 README，也能快速理解作者定位与主要项目内容。
  3. README 与主页在内容结构上保持同源，后续维护不需要分别重写两套割裂内容。
