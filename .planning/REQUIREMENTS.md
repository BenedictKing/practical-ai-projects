# Requirements: My Best Practice

**Defined:** 2026-04-17
**Core Value:** 用清晰、可信、可快速浏览的方式，让访客理解我如何通过具体项目解决普通用户在使用 AI 时遇到的真实问题。

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Positioning

- [ ] **POSI-01**: 访客进入首页后，可以在首屏快速理解站点定位为“专注解决 AI 使用中的实际阻碍”
- [ ] **POSI-02**: 首页能够同时对开源访客、潜在客户和招聘方传达“我做过什么”
- [ ] **POSI-03**: 首页明确引导访客继续查看具体项目

### Homepage

- [ ] **HOME-01**: 站点提供一个可访问的单页主页作为统一展示入口
- [ ] **HOME-02**: 首页包含自我介绍或个人定位模块
- [ ] **HOME-03**: 首页包含 AI 使用痛点总结模块
- [ ] **HOME-04**: 首页包含项目展示模块
- [ ] **HOME-05**: 首页包含方法论或设计原则模块
- [ ] **HOME-06**: 首页包含 GitHub / 文档入口模块
- [ ] **HOME-07**: 首页信息结构以清晰表达为优先，不依赖复杂视觉动效

### Projects

- [ ] **PROJ-01**: 首页能够展示多个项目卡片或区块
- [ ] **PROJ-02**: 每个项目展示项至少说明面向谁
- [ ] **PROJ-03**: 每个项目展示项至少说明解决了什么痛点
- [ ] **PROJ-04**: 每个项目展示项至少提供 GitHub 地址或在线访问地址

### README

- [ ] **RDME-01**: 仓库提供一个 README 框架，作为主页内容的精简版
- [ ] **RDME-02**: README 能够独立让 GitHub 访客快速理解你的定位与主要项目内容
- [ ] **RDME-03**: README 内容结构与主页保持同源，减少重复维护

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Content Management

- **CMGT-01**: 通过结构化内容源自动生成主页与 README
- **CMGT-02**: 自动从多个仓库抓取项目元数据并更新展示内容

### Project Detail

- **PDET-01**: 每个项目有独立详情页
- **PDET-02**: 项目详情页展示更完整的背景、设计过程与实现说明

### Interaction

- **INTA-01**: 站点支持访客互动能力（如评论、反馈或联系表单）
- **INTA-02**: 站点支持登录或后台内容管理

## Out of Scope

| Feature | Reason |
|---------|--------|
| 后台或管理面板 | v1 只验证对外展示框架，不构建内容管理系统 |
| 登录、评论、互动系统 | 当前目标是让访客理解定位并查看项目，不是建设互动平台 |
| 自动从仓库抓取项目数据 | 当前阶段先确保展示结构清晰，再考虑自动化维护 |
| 复杂视觉动效或炫技设计 | 当前优先保证表达清晰与信息可读性 |
| 多页项目详情架构 | v1 先采用单页主页，降低复杂度并快速形成成品 |
