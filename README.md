# My Best Practice

用清晰、可信、可快速浏览的方式，让访客理解我如何通过具体项目解决普通用户在使用 AI 时遇到的真实问题。

这是一个围绕我已开发项目建立的统一展示主页仓库。当前 v1 先采用单页主页结构，重点不是视觉炫技，而是先讲清楚：我是谁、我在解决什么问题、我做过哪些项目、我如何判断一个 AI 方案是否真正有效。

## 我是谁

我在把分散的 AI 实践，整理成可以被理解和复用的解决方案。

我做过的项目并不只是在“接入一个模型”或“展示一个功能”，而是围绕普通用户真正会卡住的地方：不知道从哪里开始、不知道结果是否可信、不知道流程能否持续使用。

我优先关注：

- AI 在真实工作流里的可用性，而不是演示效果
- 把复杂能力拆成普通用户能理解、能执行、能验证的步骤
- 让每个项目都能清楚回答“面向谁、解决什么、为什么值得继续看”

## 我在解决什么问题

很多 AI 产品的问题，不是能力不够，而是普通人很难稳定用起来。

- **入口太复杂，第一次使用就容易中断**  
  用户还没体会到价值，就已经被配置、选择和术语劝退。
- **结果看起来聪明，但缺少可信的判断依据**  
  用户不知道什么时候该相信输出，什么时候该回退到手动处理。
- **流程能演示，却很难融入真实场景**  
  一旦离开演示环境，效率、维护成本和协作体验都会暴露问题。

## 项目

我主要用 GitHub 作为继续浏览入口。每个项目都会先说明它面向谁、在解决什么问题，方便你快速判断是否值得继续看。

### [CCX](https://github.com/BenedictKing/ccx)

一个统一 Claude、OpenAI Chat / Codex Responses 与 Gemini 的高性能 AI API 代理和路由网关。

- **面向谁**：需要统一接入多个 AI 提供方、做路由编排、密钥管理与故障切换的开发者或团队
- **解决什么**：把分散的模型协议、鉴权方式和通道调度整合成一个统一入口，降低多上游接入与运维复杂度

### [kiro.rs](https://github.com/BenedictKing/kiro.rs)

一个用 Rust 编写的 Anthropic Claude API 兼容代理服务，把 Claude API 请求转换为 Kiro API 请求。

- **面向谁**：想让现有 Claude 生态工具继续工作，但实际接入目标是 Kiro API 的开发者
- **解决什么**：处理协议兼容、Token 刷新、多凭据容错与代理配置，让接入链路更稳定、迁移成本更低

### [Exa Search](https://github.com/BenedictKing/exa-search)

一个给 Claude Code 使用的语义搜索技能，支持 embeddings 检索、相似内容发现和结构化研究。

- **面向谁**：需要做语义检索、相似页面发现或更精细研究任务的 Claude Code 用户
- **解决什么**：弥补关键词搜索难以覆盖语义意图的问题，让研究型搜索结果更贴近真实任务

### [Codex Review](https://github.com/BenedictKing/codex-review)

一个面向 Claude Code 的专业代码审查技能，能自动收集变更上下文、生成 CHANGELOG 并联动 Codex Review。

- **面向谁**：希望把代码审查更稳定地接入日常 Git 工作流的开发者与团队
- **解决什么**：减少审查上下文缺失、变更说明不完整和人工补 CHANGELOG 的重复劳动，提高 review 质量

### [Context7 Auto Research](https://github.com/BenedictKing/context7-auto-research)

一个自动为 Claude Code 拉取最新库和框架文档的技能，用来减少“模型知道得不够新”的问题。

- **面向谁**：经常问框架、SDK 或 API 用法，且需要基于最新文档回答问题的开发者
- **解决什么**：把手动查文档的步骤自动化，减少因知识过时导致的错误实现和来回确认

### [Firecrawl Scraper](https://github.com/BenedictKing/firecrawl-scraper)

一个给 Claude Code 使用的网页抓取技能，支持内容提取、截图、PDF 解析、页面交互和整站爬取。

- **面向谁**：需要从网页、文档或复杂页面交互里提取结构化内容的开发者与研究者
- **解决什么**：把抓取、截图、交互和格式转换合并到一个技能里，减少碎片化工具切换成本

### [Tavily Web](https://github.com/BenedictKing/tavily-web)

一个面向 Claude Code 的网页搜索与研究技能，覆盖搜索、提取、爬取、站点映射和结构化调研。

- **面向谁**：需要在一个工具里完成网页搜索、信息提取与研究任务的开发者和分析者
- **解决什么**：把搜索、抓取和研究串成一条连续流程，减少从“找到信息”到“整理结论”的中间损耗

## 我如何判断方案是否有效

我会用三个原则，约束每一个 AI 方案的设计与落地：

- **先让人看懂，再让系统变强**
- **先解决重复出现的阻碍，再扩展功能范围**
- **先保证结果可判断，再追求流程自动化**
